Meta Pixel and Conversion API (CAPI)

Pixel Tracking Flow

flowchart LR
User --> Browser
Browser --> Pixel
Pixel --> MetaServer
MetaServer --> AdOptimization

Pixel is browser-based tracking. It fires when the website loads or user performs an action.

Problem After iOS14

Browsers block cookies → pixel misses conversions → ads become inefficient.

Conversion API (CAPI)

CAPI is server-to-server tracking.

flowchart LR
User --> WebsiteServer
WebsiteServer --> MetaServer
MetaServer --> Attribution
Pixel = Browser tracking CAPI = Server tracking

Why Pixel + CAPI Together?

  • Pixel catches browser events
  • CAPI catches blocked events
  • Improves attribution accuracy

Example

User buys product:

  • Pixel blocked by browser
  • CAPI sends purchase from server
  • Meta records conversion correctly

Event Matching

Meta matches user using:

  • Email
  • Phone number
  • IP address
  • User agent
